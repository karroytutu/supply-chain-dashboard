# 更新公告

## 版本 1.3.0 - 2026-03-31

### 🎉 全新功能：临过期退货管理

我们正式上线了**临过期退货管理**模块，帮助团队高效处理临期商品的退货和销售流程，做到每一件临期商品都有人跟、有人管、不遗漏。

**一句话介绍：** 系统每天自动识别临期退货商品，按流程分配给采购、仓储、营销各环节负责人，全程跟踪直到处理完成。

---

### 一、功能概览

**1. 退货单全流程管理**

退货商品从入库到处理完成，系统提供清晰的流程跟踪：

```
临期商品退货入库
      ↓
  【待确认】采购主管审核是否可退货
      ↓
  ┌── 可退货 ──┐          ┌── 不可退货 ──┐
  ↓            │          ↓              │
【待填ERP】    │       【待营销销售】     │
  ↓            │          ↓              │
【待仓储退货】 │       系统自动判定       │
  ↓            │          ↓              │
【已完成】─────┘       【已完成】────────┘
```

| 处理阶段 | 负责人 | 需要做什么 |
|----------|--------|------------|
| 待确认 | 采购主管 | 审核商品是否可以采购退货 |
| 待填ERP退货单 | 采购主管 | 在ERP系统创建退货单后，回到系统填写ERP退货单号 |
| 待仓储退货 | 仓储主管 | 安排商品退出，并在系统中录入实际退货数量 |
| 待营销销售 | 责任营销师 | 不可退货的商品，在过期前寻找渠道完成销售 |
| 已完成 | - | 退货或销售处理完毕 |

**2. 商品退货规则管理**

- 按商品设置「可采购退货」或「不可采购退货」规则
- 同一商品只需确认一次，后续同商品退货自动按规则处理，无需重复审核
- 支持批量设置，大幅减少重复操作

**3. 自动临期识别**

系统根据商品保质期自动判断是否临期，无需人工筛选：

| 商品保质期 | 临期标准 |
|-----------|----------|
| 3个月以内 | 剩余30天即为临期 |
| 3-5个月 | 剩余45天即为临期 |
| 5-9个月 | 剩余60天即为临期 |
| 9个月以上 | 剩余90天即为临期 |

**4. 智能提醒通知（钉钉推送）**

系统会在关键节点自动发送钉钉通知，确保每个环节及时处理：

| 通知时机 | 通知对象 | 通知内容 |
|----------|----------|----------|
| 每天早上 8:30 | 采购主管 | 今日新增的临期退货清单 |
| 每天早上 8:30 | 采购主管 | 尚未填写ERP退货单号的退货清单 |
| 确认不可退货时 | 责任营销师 | 需要尽快销售的临期商品 |
| 填写ERP退货单后 | 仓储主管 | 需要安排退货的商品清单 |

**5. 自动完成判定**

对于需要营销销售的商品，系统会自动监测库存变化，当残次品库存清零时自动标记为「已完成」，无需手动操作。

---

### 二、页面入口

在左侧导航栏进入 **采购管理 → 临过期退货**，包含两个页面：

- **退货单列表**：查看和处理所有临期退货单
- **采购退货规则**：管理商品的退货规则设置

---

### 三、退货单列表 — 使用说明

#### 页面布局

打开退货单列表后，页面从上到下依次是：

1. **统计卡片**（5个）：分别显示「待确认」「待填ERP」「待仓储退货」「待营销销售」「已完成」的数量，**点击任意卡片可快速筛选**该状态的退货单
2. **搜索栏**：支持按退货单号/商品名称搜索，支持按日期范围筛选
3. **退货单表格**：展示所有退货单信息

#### 表格说明

表格展示以下信息：

| 列名 | 说明 |
|------|------|
| 退货单号 | 系统自动生成的唯一编号 |
| 商品名称 | 退货商品名称 |
| 数量 | 退货数量及单位 |
| 当前库存 | 该商品实时库存数量，显示「已清零」表示库存已消耗完毕 |
| 当前剩余保质期 | 实时计算的剩余天数，颜色标记紧急程度 |
| 当前节点 | 退货单当前处理状态（彩色标签） |
| 操作 | 根据状态显示不同的操作按钮 |

**剩余保质期颜色含义：**
- 🔴 红色：≤7天（紧急）
- 🟠 橙色：8-15天（警告）
- 🟡 黄色：16-30天（关注）
- 🟢 绿色：>30天（正常）

**查看更多信息：** 点击表格每行左侧的「+」号可展开查看：生产日期、退货时间、退货时保质期（历史快照）、责任人等详细信息。

#### 操作说明

**场景一：批量审核新入库的临期商品（待确认状态）**

1. 点击顶部「待确认」统计卡片，筛选出所有待审核的退货单
2. 页面会出现操作引导提示和批量操作栏
3. 勾选需要审核的退货单（可点击「全选」选中当前页全部）
4. 根据审核结果：
   - 点击「**批量确认可退货**」→ 选中的退货单进入「待填ERP」阶段
   - 点击「**批量确认不可退货**」→ 选中的退货单进入「待营销销售」阶段，系统自动通知责任营销师

**场景二：填写ERP退货单号（待填ERP状态）**

1. 点击顶部「待填ERP」统计卡片
2. 找到需要处理的退货单，点击操作列的「**编辑**」按钮
3. 弹窗中会显示退货单号、商品名称、数量等信息（只读）
4. 在「**ERP退货单号**」输入框中填写从ERP系统获取的采购退货单号
5. 点击「**提交**」→ 退货单进入「待仓储退货」阶段，系统自动通知仓储主管

> 💡 如果发现退货单审核有误，可以点击「**回退**」按钮，将退货单退回到「待确认」状态重新审核。

**场景三：仓储执行退货（待仓储退货状态）**

1. 点击顶部「待仓储退货」统计卡片
2. 找到需要处理的退货单，点击操作列的「**执行**」按钮
3. 弹窗中会显示退货单号、商品名称、ERP退货单号等信息（只读）
4. 在「**实际退货数量**」输入框中填写仓库实际退回的数量
5. （可选）在「**备注**」框中填写退货执行的补充说明
6. 点击「**确认执行**」→ 退货单进入「待营销销售」阶段

**场景四：查看营销销售进度（待营销销售状态）**

1. 点击顶部「待营销销售」统计卡片
2. 查看需要营销部门处理的退货商品
3. 当对应商品的残次品库存清零时，系统会**自动**将退货单标记为「已完成」

> 💡 如需重新审核，可以点击「**回退**」按钮将退货单退回「待确认」状态。

### 四、采购退货规则 — 使用说明

#### 页面布局

打开采购退货规则页后，页面从上到下依次是：

1. **统计卡片**（2个）：分别显示「可采购退货」和「不可采购退货」的商品数量，**点击可快速筛选**
2. **搜索栏**：支持按商品名称/商品ID搜索，支持按规则类型筛选
3. **规则表格**：展示所有已设置规则的商品

#### 表格说明

| 列名 | 说明 |
|------|------|
| 商品名称 | 商品名称 |
| 商品ID | 商品在系统中的编号 |
| 退货规则 | ✓ 可退货（绿色）/ ✗ 不可退货（红色） |
| 确认人 | 最后设置/修改此规则的人员 |
| 确认时间 | 规则最后修改的时间 |
| 操作 | 「调整」按钮 |

#### 操作说明

**单条调整规则：**

1. 找到需要调整的商品，点击操作列的「**调整**」按钮
2. 弹窗中选择新的退货规则：「可采购退货」或「不可采购退货」
3. （可选）在「备注」框中填写调整原因
4. 点击「**确定**」保存

**批量调整规则：**

1. 在表格中勾选需要调整的商品（可逐项选择或全选）
2. 页面顶部会显示「已选中 N 条记录」和批量操作按钮
3. 点击「**批量设为可采购退货**」或「**批量设为不可采购退货**」
4. 系统批量更新后，表格自动刷新

> 💡 规则设置后，后续同一商品再次临期入库时，系统会自动根据规则决定流程走向，无需重复确认。

### 五、各角色操作指南

#### 采购主管 — 每日工作流程

1. **查看钉钉通知**：每天早上 8:30 会收到新增临期退货提醒和待填ERP退货单提醒
2. **审核退货单**：进入「退货单列表」→ 点击「待确认」卡片 → 勾选退货单 → 批量确认可退货或不可退货
3. **设置商品规则**（首次遇到的商品）：进入「采购退货规则」→ 为新商品设置退货规则
4. **填写ERP单号**：在ERP系统创建退货单后 → 回到系统点击「待填ERP」→ 逐单填写ERP退货单号

#### 仓储主管 — 操作流程

1. **查看钉钉通知**：采购主管填写ERP退货单号后，会收到待退货通知
2. **执行退货**：进入「退货单列表」→ 点击「待仓储退货」卡片 → 逐单点击「执行」→ 填写实际退货数量和备注 → 确认执行

#### 营销师 — 操作流程

1. **查看钉钉通知**：商品被确认为不可采购退货时，会收到通知
2. **跟进销售**：进入「退货单列表」→ 点击「待营销销售」卡片 → 查看需要销售的临期商品清单
3. **无需手动操作完成**：当商品的残次品库存清零后，系统自动标记为「已完成」

### 六、常见问题

**Q：退货单审核错了怎么办？**
A：在「待填ERP」或「待营销销售」状态下，点击操作列的「回退」按钮，可以将退货单退回「待确认」状态重新审核。

**Q：页面显示"暂无数据"？**
A：请确认您的账号拥有退货管理相关权限。如无权限，请联系系统管理员分配。

**Q：为什么有些商品没有出现在退货单列表中？**
A：系统仅识别符合临期标准的退货商品。不同保质期的商品有不同的临期阈值（见上方"自动临期识别"说明）。

**Q：「已完成」状态是谁操作的？**
A：「已完成」由系统自动判定——当营销销售的商品残次品库存降为0时，系统会自动标记完成，无需人工操作。

**Q：钉钉通知没有收到？**
A：请确认您的钉钉账号已与系统绑定。如未绑定，请联系系统管理员。
