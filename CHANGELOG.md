### 🎉 全新功能：应收账款管理

### 二、催收管理流程（重点！）

#### 2.1 完整催收流程图

从应收单据到期到催收完成的全流程如下：

```
应收单据到期
      ↓
系统自动生成催收任务
      ↓
营销师/催收员收到钉钉通知
      ↓
联系客户确认还款情况
      ↓
┌────────────┬────────────┬────────────┬────────────┐
↓            ↓            ↓            ↓            │
客户确认    营销担保     已回款       催收困难       │
 延期        延期       /核销         升级         │
  ↓          ↓           ↓            ↓           │
选择延期   选择延期    填写回款    填写困难     │
 日期      日期+签名    日期+凭证    原因+材料     │
  ↓          ↓           ↓            ↓           │
  └──────────┴───────────┴────────────┘           │
              ↓                                   │
        提交催收结果                              │
              ↓                                   │
     ┌── 财务审核 ──┐                             │
     ↓              ↓                             │
   通过           拒绝                            │
     ↓              ↓                             │
  出纳核实     退回重新催收 ──────────────────────┘
     ↓
  核实通过 → 催收完成
```

#### 2.2 流程阶段说明

| 阶段 | 负责人 | 需要做什么 | 操作位置 |
|------|--------|------------|----------|
| 任务生成 | 系统 | 自动识别到期未回款项，生成催收任务 | 自动执行 |
| 催收执行 | 营销师/催收员 | 联系客户，选择催收结果类型并提交 | 催收工作台 → 我的催收 |
| 财务审核 | 财务人员 | 审核催收结果的真实性 | 催收工作台 → 待审核 |
| 出纳核实 | 出纳 | 确认回款是否到账 | 催收工作台 → 待审核 |
| 催收完成 | 系统 | 标记任务完成，更新应收状态 | 自动执行 |

#### 2.3 应收状态流转

应收账款从正常到最终处理的状态变化如下：

```
正常 → 预警(5天) → 预警(2天) → 逾期 → 催收中 → 已解决/已核销/已升级
```

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| 正常 | 应收账款在账期内 | 距到期日 > 5 天 |
| 预警(5天) | 即将到期提醒 | 距到期日 ≤ 5 天 |
| 预警(2天) | 紧急到期提醒 | 距到期日 ≤ 2 天 |
| 逾期 | 已超过付款期限 | 已过到期日 |
| 催收中 | 正在进行催收处理 | 催收任务已分配 |
| 已解决 | 客户已回款或确认延期 | 催收完成并通过审核 |
| 已核销 | 坏账核销处理 | 财务确认核销 |
| 已升级 | 催收困难需上级处理 | 提交升级申请并审核通过 |

| 管理员 | 可见所有标签页，监控整体催收进度 |

#### 5.2 待办统计卡片

**点击任意卡片可快速筛选**对应状态的催收任务。

#### 5.3 我的催收 — 操作说明

营销师/催收员处理催收任务的操作步骤：

**第一步：选择催收任务**

在「我的催收」标签页中，找到需要处理的任务，点击操作列的「**催收**」按钮。

**第二步：联系客户**

弹窗中显示客户信息、应收金额、到期日期等，请根据显示的联系信息联系客户确认还款情况。

**第三步：选择催收结果**

根据客户反馈，选择以下四种处理方式之一：

| 催收结果 | 适用场景 | 需要填写的内容 |
|----------|----------|----------------|
| **客户确认延期** | 客户承认欠款，承诺延期还款 | 选择延期还款日期 |
| **营销担保延期** | 营销师为客户担保延期还款 | 选择延期日期 + 电子签名确认 |
| **已回款/核销** | 客户已付款或确认核销 | 填写回款日期 + 上传回款凭证（可选） |
| **催收困难升级** | 客户拒绝配合或无法联系 | 填写困难原因 + 上传相关材料 |

**第四步：提交审核**

填写完成后点击「**提交**」，催收结果将进入「待审核」状态，等待财务人员或出纳核实。

> 💡 提交后可在「待审核」标签页查看审核进度。如被拒绝，需要重新进行催收。

#### 5.4 待审核 — 操作说明

财务人员和出纳审核催收结果的操作步骤：

**财务审核流程：**

1. 在「待审核」标签页找到待审核的催收结果
2. 点击操作列的「**审核**」按钮
3. 查看催收详情：客户信息、催收结果类型、延期日期、凭证材料等
4. 核实催收结果的真实性后选择：
   - 点击「**通过**」→ 进入出纳核实环节（如已回款）
   - 点击「**拒绝**」→ 填写拒绝原因 → 任务退回催收员重新处理

**出纳核实流程：**

1. 对财务已通过的「已回款」催收结果进行核实
2. 点击操作列的「**核实**」按钮
3. 确认款项是否到账
4. 选择：
   - 点击「**确认到账**」→ 催收任务完成
   - 点击「**未到账**」→ 填写原因 → 任务退回催收员重新处理

### 六、考核管理 — 使用说明

#### 6.1 考核触发机制

系统会对催收超时的情况自动触发考核，**无需人工操作**：

```
催收任务分配
      ↓
开始计时
      ↓
┌──────────────────────────────────────────┐
│  0-3天  │  3-4天  │  4-7天  │  7天以上  │
│  无考核 │ 基础考核│ 加倍考核│ 全额考核 │
└──────────────────────────────────────────┘
      ↓
系统自动计算考核金额
      ↓
生成考核记录
```

#### 6.2 考核规则说明

考核金额根据超时天数和责任人角色计算：

**营销师/财务人员考核标准：**

| 超时天数 | 考核比例 | 说明 |
|----------|----------|------|
| 0-3天 | 无考核 | 正常处理时间内 |
| 3-4天 | 10% | 基础考核 |
| 4-7天 | 20% | 加倍考核 |
| 7天以上 | 50% | 全额考核 |

**营销主管考核标准：**

| 超时天数 | 考核比例 | 说明 |
|----------|----------|------|
| 0-3天 | 无考核 | 正常处理时间内 |
| 3-4天 | 15% | 基础考核 |
| 4-7天 | 30% | 加倍考核 |
| 7天以上 | 60% | 全额考核 |

> ⚠️ 考核金额 = 超时应收金额 × 考核比例

#### 6.3 考核记录表格

考核管理页面展示所有考核记录：

| 列名 | 说明 |
|------|------|
| 催收任务 | 关联的催收任务编号 |
| 客户名称 | 应收账款客户 |
| 应收金额 | 该笔应收的金额 |
| 责任人 | 被考核人员姓名 |
| 责任人角色 | 营销师/营销主管/财务 |
| 超时天数 | 从任务分配到完成的天数 |
| 考核比例 | 根据超时天数确定的考核比例 |
| 考核金额 | 实际扣除的考核金额 |
| 考核时间 | 生成考核记录的时间 |

### 七、各角色操作指南

#### 营销师/催收员 — 每日工作流程

1. **查看钉钉通知**：每天早上会收到催收任务提醒
2. **处理催收任务**：进入「催收工作台」→「我的催收」→ 选择任务 → 联系客户 → 提交催收结果
3. **跟进审核进度**：在「待审核」查看已提交催收结果的审核状态
4. **处理拒绝任务**：被拒绝的任务会退回「我的催收」，需要重新联系客户并提交

#### 营销主管 — 每日工作流程

1. **查看钉钉通知**：收到催收任务提醒和升级申请通知
2. **处理催收任务**：同营销师流程
3. **审核升级申请**：进入「待审核」→ 审核催收困难的升级申请

#### 财务人员 — 审核流程

1. **查看钉钉通知**：收到待审核催收结果提醒
2. **审核催收结果**：进入「催收工作台」→「待审核」→ 逐条审核催收结果的真实性
3. **通过或拒绝**：根据核实情况选择通过或拒绝

#### 出纳 — 核实流程

1. **查看钉钉通知**：收到待核实回款提醒
2. **核实回款**：进入「催收工作台」→「待审核」→ 核实款项是否到账
3. **确认结果**：确认到账后，催收任务完成

#### 管理员 — 监控流程

1. **查看应收总览**：监控整体应收账款情况
2. **检查催收进度**：查看各营销师的催收任务完成情况
3. **查看考核记录**：进入「考核管理」查看超时考核情况

### 八、常见问题

**Q：催收任务是如何分配的？**
A：系统根据应收账款的责任营销自动分配催收任务。如果客户没有指定责任营销，则分配给默认催收员处理。

**Q：催收结果提交后多久需要审核？**
A：财务人员和出纳应在 2 个工作日内完成审核。超时未审核的，系统会发送提醒通知。

**Q：催收被拒绝后怎么办？**
A：被拒绝的催收任务会退回到「我的催收」，您需要重新联系客户并提交新的催收结果。

**Q：什么情况下选择「催收困难升级」？**
A：以下情况可选择升级：客户拒绝配合、多次联系不上、客户经营异常等。提交后需要上传相关证明材料。

**Q：考核金额可以申诉吗？**
A：可以。请联系财务部门说明情况，由管理员核实后可调整考核记录。

**Q：页面显示"暂无数据"？**
A：请确认您的账号拥有应收账款管理相关权限。如无权限，请联系系统管理员分配。

**Q：钉钉通知没有收到？**
A：请确认您的钉钉账号已与系统绑定。如未绑定，请联系系统管理员。

## 版本 1.3.0 - 2026-03-31

### 🎉 全新功能：临过期退货管理

我们正式上线了**临过期退货管理**模块，帮助团队高效处理临期商品的退货和销售流程，做到每一件临期商品都有人跟、有人管、不遗漏。

**一句话介绍：** 系统每天自动识别临期退货商品，按流程分配给采购、仓储、营销各环节负责人，全程跟踪直到处理完成。

---

### 一、功能概览

**1. 退货单全流程管理**

退货商品从入库到处理完成，系统提供清晰的流程跟踪：

```
临期商品退货入库
      ↓
  【待确认】采购主管审核是否可退货
      ↓
  ┌── 可退货 ──┐          ┌── 不可退货 ──┐
  ↓            │          ↓              │
【待填ERP】    │       【待营销销售】     │
  ↓            │          ↓              │
【待仓储退货】 │       系统自动判定       │
  ↓            │          ↓              │
【已完成】─────┘       【已完成】────────┘
```

| 处理阶段 | 负责人 | 需要做什么 |
|----------|--------|------------|
| 待确认 | 采购主管 | 审核商品是否可以采购退货 |
| 待填ERP退货单 | 采购主管 | 在ERP系统创建退货单后，回到系统填写ERP退货单号 |
| 待仓储退货 | 仓储主管 | 安排商品退出，并在系统中录入实际退货数量 |
| 待营销销售 | 责任营销师 | 不可退货的商品，在过期前寻找渠道完成销售 |
| 已完成 | - | 退货或销售处理完毕 |

**2. 商品退货规则管理**

- 按商品设置「可采购退货」或「不可采购退货」规则
- 同一商品只需确认一次，后续同商品退货自动按规则处理，无需重复审核
- 支持批量设置，大幅减少重复操作

**3. 自动临期识别**

系统根据商品保质期自动判断是否临期，无需人工筛选：

| 商品保质期 | 临期标准 |
|-----------|----------|
| 3个月以内 | 剩余30天即为临期 |
| 3-5个月 | 剩余45天即为临期 |
| 5-9个月 | 剩余60天即为临期 |
| 9个月以上 | 剩余90天即为临期 |

**4. 智能提醒通知（钉钉推送）**

系统会在关键节点自动发送钉钉通知，确保每个环节及时处理：

| 通知时机 | 通知对象 | 通知内容 |
|----------|----------|----------|
| 每天早上 8:30 | 采购主管 | 今日新增的临期退货清单 |
| 每天早上 8:30 | 采购主管 | 尚未填写ERP退货单号的退货清单 |
| 确认不可退货时 | 责任营销师 | 需要尽快销售的临期商品 |
| 填写ERP退货单后 | 仓储主管 | 需要安排退货的商品清单 |

**5. 自动完成判定**

对于需要营销销售的商品，系统会自动监测库存变化，当残次品库存清零时自动标记为「已完成」，无需手动操作。

在左侧导航栏进入 **采购管理 → 临过期退货**，包含两个页面：

- **退货单列表**：查看和处理所有临期退货单
- **采购退货规则**：管理商品的退货规则设置

### 三、退货单列表 — 使用说明

#### 页面布局

打开退货单列表后，页面从上到下依次是：

1. **统计卡片**（5个）：分别显示「待确认」「待填ERP」「待仓储退货」「待营销销售」「已完成」的数量，**点击任意卡片可快速筛选**该状态的退货单
2. **搜索栏**：支持按退货单号/商品名称搜索，支持按日期范围筛选
3. **退货单表格**：展示所有退货单信息

#### 表格说明

表格展示以下信息：

| 列名 | 说明 |
|------|------|
| 退货单号 | 系统自动生成的唯一编号 |
| 商品名称 | 退货商品名称 |
| 数量 | 退货数量及单位 |
| 当前库存 | 该商品实时库存数量，显示「已清零」表示库存已消耗完毕 |
| 当前剩余保质期 | 实时计算的剩余天数，颜色标记紧急程度 |
| 当前节点 | 退货单当前处理状态（彩色标签） |
| 操作 | 根据状态显示不同的操作按钮 |

**剩余保质期颜色含义：**
- 🔴 红色：≤7天（紧急）
- 🟠 橙色：8-15天（警告）
- 🟡 黄色：16-30天（关注）
- 🟢 绿色：>30天（正常）

**查看更多信息：** 点击表格每行左侧的「+」号可展开查看：生产日期、退货时间、退货时保质期（历史快照）、责任人等详细信息。

#### 操作说明

**场景一：批量审核新入库的临期商品（待确认状态）**

1. 点击顶部「待确认」统计卡片，筛选出所有待审核的退货单
2. 页面会出现操作引导提示和批量操作栏
3. 勾选需要审核的退货单（可点击「全选」选中当前页全部）
4. 根据审核结果：
   - 点击「**批量确认可退货**」→ 选中的退货单进入「待填ERP」阶段
   - 点击「**批量确认不可退货**」→ 选中的退货单进入「待营销销售」阶段，系统自动通知责任营销师

**场景二：填写ERP退货单号（待填ERP状态）**

1. 点击顶部「待填ERP」统计卡片
2. 找到需要处理的退货单，点击操作列的「**编辑**」按钮
3. 弹窗中会显示退货单号、商品名称、数量等信息（只读）
4. 在「**ERP退货单号**」输入框中填写从ERP系统获取的采购退货单号
5. 点击「**提交**」→ 退货单进入「待仓储退货」阶段，系统自动通知仓储主管

> 💡 如果发现退货单审核有误，可以点击「**回退**」按钮，将退货单退回到「待确认」状态重新审核。

**场景三：仓储执行退货（待仓储退货状态）**

1. 点击顶部「待仓储退货」统计卡片
2. 找到需要处理的退货单，点击操作列的「**执行**」按钮
3. 弹窗中会显示退货单号、商品名称、ERP退货单号等信息（只读）
4. 在「**实际退货数量**」输入框中填写仓库实际退回的数量
5. （可选）在「**备注**」框中填写退货执行的补充说明
6. 点击「**确认执行**」→ 退货单进入「待营销销售」阶段

**场景四：查看营销销售进度（待营销销售状态）**

1. 点击顶部「待营销销售」统计卡片
2. 查看需要营销部门处理的退货商品
3. 当对应商品的残次品库存清零时，系统会**自动**将退货单标记为「已完成」

> 💡 如需重新审核，可以点击「**回退**」按钮将退货单退回「待确认」状态。

### 四、采购退货规则 — 使用说明

#### 页面布局

打开采购退货规则页后，页面从上到下依次是：

1. **统计卡片**（2个）：分别显示「可采购退货」和「不可采购退货」的商品数量，**点击可快速筛选**
2. **搜索栏**：支持按商品名称/商品ID搜索，支持按规则类型筛选
3. **规则表格**：展示所有已设置规则的商品

#### 表格说明

| 列名 | 说明 |
|------|------|
| 商品名称 | 商品名称 |
| 商品ID | 商品在系统中的编号 |
| 退货规则 | ✓ 可退货（绿色）/ ✗ 不可退货（红色） |
| 确认人 | 最后设置/修改此规则的人员 |
| 确认时间 | 规则最后修改的时间 |
| 操作 | 「调整」按钮 |

#### 操作说明

**单条调整规则：**

1. 找到需要调整的商品，点击操作列的「**调整**」按钮

3. （可选）在「备注」框中填写调整原因
4. 点击「**确定**」保存

**批量调整规则：**

1. 在表格中勾选需要调整的商品（可逐项选择或全选）
2. 页面顶部会显示「已选中 N 条记录」和批量操作按钮
3. 点击「**批量设为可采购退货**」或「**批量设为不可采购退货**」
4. 系统批量更新后，表格自动刷新

> 💡 规则设置后，后续同一商品再次临期入库时，系统会自动根据规则决定流程走向，无需重复确认。

### 五、各角色操作指南

#### 采购主管 — 每日工作流程

1. **查看钉钉通知**：每天早上 8:30 会收到新增临期退货提醒和待填ERP退货单提醒
2. **审核退货单**：进入「退货单列表」→ 点击「待确认」卡片 → 勾选退货单 → 批量确认可退货或不可退货
3. **设置商品规则**（首次遇到的商品）：进入「采购退货规则」→ 为新商品设置退货规则
4. **填写ERP单号**：在ERP系统创建退货单后 → 回到系统点击「待填ERP」→ 逐单填写ERP退货单号

#### 仓储主管 — 操作流程

1. **查看钉钉通知**：采购主管填写ERP退货单号后，会收到待退货通知
2. **执行退货**：进入「退货单列表」→ 点击「待仓储退货」卡片 → 逐单点击「执行」→ 填写实际退货数量和备注 → 确认执行

#### 营销师 — 操作流程

1. **查看钉钉通知**：商品被确认为不可采购退货时，会收到通知
2. **跟进销售**：进入「退货单列表」→ 点击「待营销销售」卡片 → 查看需要销售的临期商品清单
3. **无需手动操作完成**：当商品的残次品库存清零后，系统自动标记为「已完成」

### 六、常见问题

**Q：退货单审核错了怎么办？**
A：在「待填ERP」或「待营销销售」状态下，点击操作列的「回退」按钮，可以将退货单退回「待确认」状态重新审核。

**Q：页面显示"暂无数据"？**
A：请确认您的账号拥有退货管理相关权限。如无权限，请联系系统管理员分配。

**Q：为什么有些商品没有出现在退货单列表中？**
A：系统仅识别符合临期标准的退货商品。不同保质期的商品有不同的临期阈值（见上方"自动临期识别"说明）。

**Q：「已完成」状态是谁操作的？**
A：「已完成」由系统自动判定——当营销销售的商品残次品库存降为0时，系统会自动标记完成，无需人工操作。

**Q：钉钉通知没有收到？**
A：请确认您的钉钉账号已与系统绑定。如未绑定，请联系系统管理员。
