# 端口配置规范

## 端口分配

| 环境 | 前端 | 后端 API | 说明 |
|------|------|----------|------|
| 生产环境 | 3000 | 8000 | 生产部署使用 |
| 开发环境 | 3100 | 8100 | 本地开发使用 |

## 强制规则

### 端口使用规范

1. **严格按端口分配表使用端口**：开发和部署必须严格按照上述端口分配表执行，禁止使用其他端口
2. **端口占用检查**：启动服务前必须检查目标端口是否被占用
3. **端口清理范围**：仅允许清理释放本规范中定义的端口（3000、8000、3100、8100），**禁止清理其他端口**，避免影响其他项目

### 端口检查与清理命令

```bash
# 检查端口占用
lsof -i :端口号

# 清理占用端口的进程（仅限本规范定义的端口）
kill -9 $(lsof -t -i :端口号)
```

### 可清理端口列表

以下端口可以执行清理操作：
- **3000** - 生产环境前端
- **8000** - 生产环境后端 API
- **3100** - 开发环境前端
- **8100** - 开发环境后端 API

## 启动命令

### 开发环境

**后端服务 (端口 8100)**
```bash
cd dev/backend
npm install  # 首次运行需要安装依赖
npm run dev
```

**前端服务 (端口 3100)**
```bash
cd dev/frontend
npm install  # 首次运行需要安装依赖
npm run dev
```

**统一启动（根目录）**
```bash
# 同时启动前端和后端
npm run dev
```

### 生产环境

**使用 Docker Compose 部署**
```bash
cd deploy
docker-compose up -d
```

**或单独构建运行**

后端服务 (端口 8000)
```bash
cd dev/backend
npm run build
# 编译产物输出到 prod/backend/dist/
```

前端服务 (端口 3000)
```bash
cd dev/frontend
npm run build
# 构建产物输出到 prod/frontend/dist/
```

## 配置文件

### 后端配置
- 开发环境配置: `dev/backend/.env.development` (端口 8100)
- 生产环境配置: `dev/backend/.env.production` (端口 8000)

### 前端配置
- 配置文件: `dev/frontend/.umirc.ts`
- 开发环境代理: `/api` -> `http://localhost:8100`
- 生产环境代理: 通过 nginx 配置 (`deploy/nginx.conf`)

## 注意事项

1. 启动服务前请先检查端口是否被占用
2. 仅能清理本规范中定义的端口（3000、8000、3100、8100），不允许清理其他项目的端口
3. 检查端口命令: `lsof -i :端口号`
4. 开发环境和生产环境使用不同端口，可以在同一台机器上同时运行
5. 切换环境时注意使用正确的启动命令

## 目录结构说明

```
项目根目录/
├── dev/                    # 开发环境（提交至 Git）
│   ├── frontend/           # 前端开发目录
│   └── backend/            # 后端开发目录
├── prod/                   # 生产环境（不提交至 Git）
│   ├── frontend/dist/      # 前端构建产物
│   └── backend/dist/       # 后端编译产物
└── deploy/                 # 部署配置
    ├── docker-compose.yml
    ├── nginx.conf
    ├── Dockerfile.frontend
    └── Dockerfile.backend
```
